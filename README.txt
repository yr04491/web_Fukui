# README.txt

## 1. 構成する要素

このWebサイトは、以下の主要なコンポーネント要素によって構成されています。

* **Layout (レイアウト)**
    * サイト全体の骨格を定義するコンポーネントです。
    * 画面サイズに応じて、デスクトップ用ナビゲーションやバナーの表示・非表示を切り替えます。
    * `Navigation` (デスクトップ用サイドバーナビ)
    * `Banner` (デスクトップ用サイドバーバナー)
    * `MainContent` / `children` (各ページのメインコンテンツ)
    * `HamburgerMenu` (モバイル用ハンバーガーメニュー)
    * `Overlay` (ハンバーガーメニュー表示時に使用されるオーバーレイ)

* **Navigation (ナビゲーション)**
    * デスクトップ表示（幅983px以上）の際に左側に表示されるサイドナビゲーションです。
    * `NavigationHeader`: ロゴとタイトル（ヘッダー部分）。
    * `NavigationItem`: 6つの主要セクションへのリンク項目。
    * `Search Section`: 「探してみよう」セクション（体験談を探す、居場所を探す など）。
    * `NavFooter`: 「プロジェクトと私たちについて」へのリンクフッター。

* **HamburgerMenu (ハンバーガーメニュー)**
    * モバイル表示（幅982px以下）で使用されるナビゲーションです。

* **MainContent (メインコンテンツ)**
    * 各ページの主要な内容を表示するエリアです。
    * デスクトップでは中央に配置されます。
    * `commonPageLayout.module.css` の `pageContainer` スタイルが共通の外枠として適用されます。

* **Banner (バナー)**
    * デスクトップ表示（幅1205px以上）の際に右側に表示されるサイドバーです。

* **Footer (フッター)**
    * 各メインコンテンツの最下部に表示される共通フッターです。
    * サイトのタイトル、ロゴ（クリックでホームへ）、フッターメニュー（運営について）が含まれます。

## 2. 画面サイズごとに想定するデザイン案

`constants.css` および `useResponsive.js` に基づくレスポンシブデザイン仕様は以下の通りです。

### ブレークポイント別レイアウト

#### 1. **1304px以上** - 中央寄せ表示
* 1304pxのレイアウト（下記2）を画面中央に配置します。
* 左右に均等な余白が追加されます。
* **レイアウト案:** `[可変余白] [ナビ 270px] [空白 72px] [メイン 540px] [空白 72px] [バナー 200px] [可変余白]`

#### 2. **1304px以下** - デフォルトレイアウト
* すべての要素（ナビ、メイン、バナー）が表示されます。
* 右側に100px以上の可変余白が設定されます。
* **レイアウト案:** `[左余白 50px] [ナビ 270px] [空白 72px] [メイン 540px] [空白 72px] [バナー 200px] [右余白 100px〜(可変)]`

#### 3. **1205px以下** - 右余白の縮小
* バナーは引き続き表示されます。
* 右側の可変余白が最小1pxまで縮小されます。
* **レイアウト案:** `[左余白 50px] [ナビ 270px] [空白 72px] [メイン 540px] [空白 72px] [バナー 200px] [右余白 1px〜(可変)]`

#### 4. **1204px以下** - バナー非表示
* **バナー (`--banner-width: 200px;`) が非表示**になります。
* メインコンテンツの右側の余白が可変になります。
* **レイアウト案:** `[左余白 50px] [ナビ 270px] [空白 72px] [メイン 540px] [右余白 (可変)]`

#### 5. **982px以下** - ナビ非表示、ハンバーガーメニュー表示
* **ナビゲーション (`--nav-width: 270px;`) が非表示**になります。
* **ハンバーガーメニュー**が画面右上に固定表示されます。
* メインコンテンツ (`--content-width: 540px;`) が左右均等の余白で中央に配置されます。
* **レイアウト案:** `[可変余白] [メイン 540px] [可変余白]`

#### 6. **560px以下** - 比率ベース縮小
* メインコンテンツが画面幅に応じた比率（幅 96.43%）で縮小されます。
* 左右の余白も比率（各 1.79%）で固定されます。
* **レイアウト案:** `[左余白 1.79%(可変)] [メイン 96.43%(可変)] [右余白 1.79%(可変)]`

### ハンバーガーメニュー仕様

* **表示条件**: 982px以下
* **位置**: 画面右上に固定
* **動作**: 画面右側からスライドイン
* **幅**:
    * **561px以上**: 270px固定
    * **560px以下**: 画面幅の 46.55%（可変）
* **最大幅**: 270px

## 3. 各ページのデザインや表示に関する説明

`App.js` で定義されたルーティングに基づき、各ページは `Layout` コンポーネントを共通の親として持ち、メインコンテンツ部分を入れ替えることで表示されます。
すべてのページコンテンツには共通の外枠スタイル (`layoutStyles.pageContainer`) が適用されます。

* **HomePage (パス: `/`)**
    * Webサイトのトップページです。
    * `MainContent` (home用) が表示されます。
    * 内容は `TitleSection` (タイトル)、`OpeningSection` (導入文)、および `Section00` から `Section05` までの各セクションへの導線（カード）で構成されます。
    * コンテンツの最後には共通の `Footer` が表示されます。

* **Section00Page (パス: `/section00`)**
    * 「まずは、どうする？」ページです。
    * `Section00Content` が表示されます。
    * ページ上部には「00」のアイコンと「まずどうする」というタイトル、および「ご家族が落ち着くことが第一です」といったサブタイトルが表示されます。
    * 内容は、不登校と引きこもりの違い（文科省・厚労省の定義引用）、8050問題、子どもへの接し方（責めない、居場所を作る）についての説明が含まれます。
    * 関連リンクへのボタン（「小中学校の卒業について」「体験談をみてみよう」）が設置されています。
    * コンテンツの最後には共通の `Footer` が表示されます。

* **Section01Page (パス: `/section01`)**
    * 「学校に相談してみよう」ページです。
    * `Section01Content` が表示されます。
    * ページ上部には「01」のアイコンと「学校に相談」というタイトル、および「まずは学校に相談してみよう」といったサブタイトルが表示されます。
    * 内容は、学校への相談の勧め、体験談、学校内の支援（スクールカウンセラー、校内サポートルーム、ライフパートナー制度）の紹介が含まれます。
    * 関連リンクへのボタンが設置されています。
    * コンテンツの最後には共通の `Footer` が表示されます。

* **Section02Page (パス: `/section02`)**
    * 「行政が行う公的支援」ページです。
    * `Section02Content` が表示されます。
    * ページ上部には「02」のアイコンと「行政が行う公的支援」というタイトル、および「学校以外で支援や相談に乗ってくれるところです。」といったサブタイトルが表示されます。
    * 内容として、公的支援の窓口紹介（例：チャレンジきょうしつ、ソーシャルワーカー）が含まれます。
    * 関連リンクへのボタンが設置されています。
    * コンテンツの最後には共通の `Footer` が表示されます。
